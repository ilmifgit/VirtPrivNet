## 1. Подготовка сервера

### 1.1. Аренда VPS
Для стабильной работы рекомендуются следующие параметры:
*   **Хостинг:** DigitalOcean, Vultr, Play2GO, Hetzner и аналоги.
*   **ОС:** Ubuntu 22.04 LTS (или новее).
*   **Минимальные характеристики:** 1 vCPU, 1–2 ГБ RAM, от 20 ГБ SSD.

> 📝 **Совет:** Обязательно сохраните **IP-адрес** и **пароль root** (или SSH-ключ), полученные от провайдера.

### 1.2. Подключение по SSH
Для начала работы откройте терминал на вашем локальном компьютере и выполните:
```bash
ssh root@ваш-ip-адрес
```
*При первом подключении система попросит подтвердить добавление хоста — введите `yes` и укажите пароль.*

## 2. Первичная настройка сервера

### 2.1. Обновление системы
Перед установкой любых компонентов приведите систему в актуальное состояние:
```bash
apt update && apt upgrade -y
```

### 2.2. Установка и настройка брандмауэра (UFW)
Безопасность начинается с настройки сетевого экрана:

```bash
# Установка UFW
apt install ufw -y

# Разрешаем SSH (стандартный порт 22)
ufw allow 22/tcp

# Открываем порты для работы сервисов
ufw allow 8443/tcp      # Порт для VPN-соединений
ufw allow 11167/tcp     # Порт для веб-панели управления
ufw allow 80/tcp        # Порт для HTTP
ufw allow 443/tcp       # Порт для HTTPS

# Активация брандмауэра
ufw enable
```

***

### 2.3. Повышение безопасности SSH (рекомендуется)
Настоятельно советуем ограничить доступ к серверу, чтобы избежать брутфорс-атак:

1. Откройте конфигурационный файл:
   ```bash
   nano /etc/ssh/sshd_config
   ```
2. Внесите следующие изменения:
   ```ssh
   Port 49222                 # Замените 22 на свой нестандартный порт
   PermitRootLogin no         # Запрет входа под root
   PasswordAuthentication no  # Вход разрешен только по SSH-ключам
   ```
3. Примените изменения:
   ```bash
   systemctl restart sshd
   ```

> ⚠️ **Важно:** Перед тем как закрыть текущую сессию, убедитесь, что вы успешно подключились к серверу через новый порт с использованием SSH-ключа. В противном случае вы можете потерять доступ к серверу.

## 3. Установка панели управления 3x-ui

### 3.1. Загрузка и установка
Выполните следующую команду в терминале:
```bash
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

### 3.2. Ответы на вопросы установщика
Во время установки следуйте рекомендациям ниже:

| Вопрос установщика | Рекомендуемый ответ |
| :--- | :--- |
| **Database Selection** | `1` (SQLite) |
| **Customize Panel Port?** | `y` |
| **Panel Port** | `11167` |
| **Username** | Ваш уникальный логин |
| **Password** | Ваш надежный пароль |
| **SNI domain** | `www.microsoft.com` (или любой популярный домен) |
| **Port for Reality** | `443` |
| **SSL Certificate** | `2` (Let's Encrypt for IP) |
| **IPv6 address** | Оставьте пустым (Enter) |
| **Email for notifications**| Оставьте пустым (Enter) |
| **Port for ACME** | Оставьте пустым (Enter) |

После успешной установки система выведет URL для входа в панель управления:
`https://ваш-ip-адрес:11167/случайный-путь`

> 💾 **Обязательно сохраните** этот URL, а также логин и пароль в надежном месте (например, в менеджере паролей).

### 3.3. Проверка работы панели
Если возникла необходимость проверить статус или перезапустить службу, используйте команды:

*   **Проверка статуса:** `x-ui status`
*   **Принудительный запуск:** `x-ui start`

## 4. Создание первого подключения (Inbound)

### 4.1. Вход в веб-панель
Откройте браузер и перейдите по URL, полученному на предыдущем шаге. 

> 🌐 **Важно:** Если браузер выдает предупреждение «Соединение не защищено», нажмите «Дополнительно» → «Перейти на сайт (небезопасно)». Это происходит из-за использования самоподписанного сертификата.

### 4.2. Создание Inbound
1. В левом меню выберите **«Inbounds»** → нажмите кнопку **«+ Add Inbound»**.
2. **Вкладка «Основное»:**
   - **Включить:** ✅ (активировано)
   - **Примечание:** `myvpn` (или удобное вам имя)
   - **Протокол:** `vless`
   - **Порт:** `8443` (убедитесь, что этот порт открыт в UFW)
3. **Вкладка «Поток»:**
   - **Транспорт:** `raw` (TCP)
   - **Flow:** `xtls-rprx-vision`
4. **Вкладка «Безопасность»:**
   - **Security:** `reality`
   - **uTLS:** `random` 
   - **Target:** `www.microsoft.com:443`
   - **SNI:** `www.microsoft.com`
   - Нажмите кнопку **«Получить новый сертификат»** — система автоматически заполнит ключи.
   - **Short IDs:** Оставьте значения по умолчанию.
5. **Вкладка «Клиенты»:**
   - Нажмите **«+ Add Client»**.
   - **Email:** `myphone` (имя пользователя)
   - **Flow:** `xtls-rprx-vision`
   - Нажмите кнопку **«Create»**.
6. **Завершение:** Внизу окна нажмите **«Add»** (или «Update»).

### 4.3. Экспорт ссылки для подключения
Теперь, когда подключение создано:
1. В списке `Inbounds` найдите созданную строку.
2. Нажмите на иконку меню (**⋮**) и выберите **«Export link»** (или иконку QR-кода).
3. Скопируйте ссылку, которая начинается с `vless://...`. 

> 💡 **Совет:** Эта ссылка является «ключом» от вашего VPN. Не передавайте её третьим лицам.

## 5. Настройка клиента

### 5.1. Установка приложения
Для работы VPN выберите клиент, подходящий для вашей операционной системы:

| Платформа | Рекомендуемое приложение |
| :--- | :--- |
| **Android** | [v2rayNG](https://github.com/2dust/v2rayNG/releases) |
| **iOS** | Streisand, FoXray, DefaultVPN, Shadowrocket |
| **Windows** | [v2rayN](https://github.com/2dust/v2rayN/releases) |
| **macOS** | V2rayU, V2Box |

### 5.2. Импорт конфигурации
Чтобы подключиться, выполните следующие действия:
1. Откройте приложение на своем устройстве.
2. Выберите опцию **«Импорт из буфера обмена»** (обычно это кнопка `+` или соответствующий пункт меню).
3. Проверьте корректность импортированных данных:
   - **Address (Адрес):** IP-адрес вашего сервера.
   - **Port (Порт):** 8443 (или тот, который вы указали в 3x-ui).
   - **Protocol (Протокол):** `vless`.
   - **Security:** `reality`.
   - **SNI:** `www.microsoft.com` (должен совпадать с настройками в панели).
   - **Flow (Поток):** `xtls-rprx-vision`.

> 💡 **Совет:** Для полной анонимности рекомендуем включить **TUN-режим** (или «System Proxy») в настройках приложения. Это направит весь интернет-трафик вашего устройства через созданный туннель.

### 5.3. Подключение
1. Нажмите кнопку **«Подключиться»** (обычно иконка «V» или кнопка питания).
2. Если статус изменился на «Подключено» (или появилось уведомление с пингом), значит, соединение установлено.
3. Проверьте работу, открыв [2ip.ru](https://2ip.ru) или [browserleaks.com/ip](https://browserleaks.com/ip) — ваш IP-адрес должен соответствовать IP-адресу вашего VPS.

## 6. Частые ошибки и их решение

При возникновении проблем обратитесь к этой таблице:

| Ошибка / Симптом | Вероятная причина | Решение |
| :--- | :--- | :--- |
| **`Connection refused`** | Порт VPN закрыт в брандмауэре. | Проверьте правила UFW: `ufw allow <порт>/tcp` (например, `ufw allow 8443/tcp`). Также убедитесь, что внешний файрвол вашего хостинг-провайдера не блокирует этот порт. |
| **`TLS handshake error`** | Неправильный `SNI` или публичный ключ, несоответствие настроек клиента и сервера. | В настройках клиента убедитесь, что `SNI` точно совпадает с тем, что указан в панели `3x-ui`. При необходимости перегенерируйте ключи в панели. |
| **Веб-панель `3x-ui` не открывается** | Порт панели (`11167`) заблокирован или служба не запущена. | Проверьте UFW: `ufw allow 11167/tcp`. Затем проверьте статус службы: `x-ui status`. Если неактивна, запустите: `x-ui start`. |
| **Клиент не подключается (нет ответа)** | Неправильно выбран протокол, транспорт или `Flow`. | В панели `3x-ui` убедитесь, что выбран протокол `vless`, транспорт `raw` (TCP), и правильный `Flow` (`xtls-rprx-vision`). |
| **Порт 443 занят** | Другой сервис (например, веб-сервер Nginx) уже использует этот порт. | Вам потребуется либо перенастроить один из сервисов на другой порт, либо использовать `SNI` для разделения трафика, если это поддерживается. Проверьте занятые порты командой: `ss -tlnp | grep :443`. |
| **DNS-домен не резолвится** | `A`-запись домена не добавлена или еще не распространилась по DNS-серверам. | Добавьте `A`-запись, указывающую на IP вашего VPS, в настройках DNS у вашего регистратора домена. Распространение может занять от нескольких минут до 24 часов. |
| **Сервер обрывает соединение (EOF)** | Несоответствие версий клиента/сервера, некорректный `fingerprint` или `UUID`. | Убедитесь, что используете актуальные версии клиента. Попробуйте изменить `fingerprint` в настройках клиента на `chrome` или `random` (если это доступно). |

## 7. Совмещение с другими сервисами

Один VPS может эффективно обслуживать несколько сервисов одновременно, если правильно распределить порты.

### Основные сервисы и рекомендуемые порты:
*   **Веб-сайт (HTTPS):** 443 (требуется для SSL-сертификатов)
*   **Защищённый туннель (VPN):** 8443 (или любой другой свободный порт, отличный от 443)
*   **Панель управления `3x-ui`:** 11167
*   **SSH:** 49222 (или любой другой нестандартный порт)

### Главное правило:
Каждый сервис должен работать на **уникальном порту**.

### Проверка занятых портов:
Перед запуском нового сервиса или при возникновении проблем с портами, используйте команду:
```bash
ss -tlnp | grep :<порт>
```
Замените `<порт>` на номер порта, который вы хотите проверить (например, `ss -tlnp | grep :443`). Эта команда покажет, какой процесс использует указанный порт.
